## Spacing Scale

Declared values — all from `AppSpacing` constants. No literal numeric values allowed.

| Token          | Dart Constant      | Value | Usage in this phase                                    |
|----------------|--------------------|-------|--------------------------------------------------------|
| xs             | `AppSpacing.xs`    | 4px   | Grid cell gaps (D-05), chip `spacing`/`runSpacing`     |
| sm             | `AppSpacing.sm`    | 8px   | Icon padding, bookmark button inset from card edge     |
| md             | `AppSpacing.md`    | 16px  | Screen body padding, bottom sheet horizontal padding   |
| lg             | `AppSpacing.lg`    | 24px  | Section headers spacing, bottom sheet vertical padding |
| xl             | `AppSpacing.xl`    | 32px  | Not used in primary layouts this phase                 |
| xxl            | `AppSpacing.xxl`   | 48px  | Border radius on circular buttons (existing pattern)   |

Exceptions:
- Grid cell main-axis and cross-axis spacing: `AppSpacing.xs` (4px). The 2px gap specified
  in D-05 is treated as "approximately 2px" — use `AppSpacing.xs` (4px) per the no-magic-numbers
  rule. If the product owner requires exactly 2px, a named constant `AppSpacing.xxs = 2` must
  be added to `spacing.dart` first.
- Touch targets for the bookmark icon button: minimum 44px (Material minimum tap target).
  `IconButton` in Flutter automatically enforces 48px minimum tap target — no override needed.
- `SliverAppBar` minimum extent: implicit from `AppBarTheme` — not a spacing override.

## Color

All colors are referenced by their `AppColors.*` constant. No hex values in implementation.

| Role              | Dart Constant              | Hex       | Usage                                                   |
|-------------------|----------------------------|-----------|---------------------------------------------------------|
| Dominant (60%)    | `AppColors.background`     | #1A1D2E   | Scaffold background, CustomScrollView background        |
| Secondary (30%)   | `AppColors.surface`        | #252840   | AppBar background, bottom sheet background              |
| Surface container | `AppColors.surfaceContainer` | #2F3250 | Grid cell background fallback (null artCrop), chip bg   |
| Accent (10%)      | `AppColors.primary`        | #C9A84C   | See reserved list below                                 |
| Accent pressed    | `AppColors.primaryVariant` | #F0A500   | Pressed/hover state on accent elements                  |
| Muted text        | `AppColors.onSurfaceMuted` | #9E9EAE   | Secondary labels, icon tints in inactive state          |
| Primary text      | `AppColors.onBackground`   | #E8E8E8   | All primary text, icon fill                             |
| Error / delete    | `AppColors.error`          | #CF6679   | Delete icon button, "Reset" text button, destructive CTA|

Accent (`AppColors.primary`) reserved exclusively for:
1. Bookmark icon fill color when card IS saved (`Icons.favorite` filled state)
2. Selected filter chip background overlay (`AppColors.primary.withValues(alpha: 0.2)`)
3. Snackbar "Undo" action label text
4. Filter bottom sheet "Apply" / confirm action label (if applicable)
5. Active preset name suffix `*` chip label (inherited from Phase 2 pattern)

Accent is NOT used for: general icon tints, nav bar items, unselected states, or body text.

Source: `lib/core/theme/app_theme.dart` — verified in codebase.

## Component Inventory

Components to implement or extend in this phase. All must use `Theme.of(context)` — no
hardcoded styles.

### 1. Bookmark Overlay Button (FAV-01, D-01, D-04)

Placed as a `Positioned` child inside the existing `Stack` in `_CardFaceWidget`
(`card_swipe_screen.dart`). Slots alongside the existing REVEAL label overlay.

| Property       | Value                                                                     |
|----------------|---------------------------------------------------------------------------|
| Position       | `bottom: AppSpacing.sm, right: AppSpacing.sm`                             |
| Icon unsaved   | `Icons.favorite_border`, color `AppColors.onBackground`                   |
| Icon saved     | `Icons.favorite`, color `AppColors.error`                                 |
| Tap unsaved    | Fires save action → shows Snackbar                                        |
| Tap saved      | `onPressed: null` (disabled — D-04, no toggle/remove from swipe screen)   |
| Size           | Default `IconButton` (48px minimum tap target enforced by Flutter)        |

### 2. Favourites Grid — FavouritesScreen (FAV-02)

`CustomScrollView` with `SliverAppBar` + `SliverGrid`.

Primary focal point: the 3-column artCrop image grid, which occupies the full viewport and draws the eye first.

| Property            | Value                                                               |
|---------------------|---------------------------------------------------------------------|
| Grid columns        | 3 (`crossAxisCount: 3`)                                             |
| Cell aspect ratio   | 1.0 (square — art crop images are square)                           |
| Cell spacing        | `mainAxisSpacing: AppSpacing.xs`, `crossAxisSpacing: AppSpacing.xs` |
| Cell image          | `CachedNetworkImage` with `artCropUrl`                              |
| Cell null fallback  | `ColoredBox(color: AppColors.surfaceContainer)` (D-06 / Pitfall 6) |
| Sort order          | Newest saved first (`savedAt` descending) — Claude's discretion     |
| App bar (default)   | Title "Favourites", action: filter icon (`Icons.filter_list`)       |
| App bar (selecting) | Title "X selected", action: delete icon (`Icons.delete`)            |

### 3. Multi-Select Grid Cell Overlay (D-06, D-07)

`Stack` with `Positioned` checkmark overlay. Visibility gated on `_selectedIds.contains(card.id)`.

| Property              | Value                                                                     |
|-----------------------|---------------------------------------------------------------------------|
| Overlay background    | `AppColors.primary.withValues(alpha: 0.3)` covering full cell             |
| Checkmark icon        | `Icons.check_circle`, color `AppColors.onBackground`, size 24px           |
| Long-press enters     | Multi-select mode, selects the long-pressed card                          |
| Tap in select mode    | Toggles selection of that card                                            |
| Tap outside mode      | Navigates to `/favourites/:id`                                            |
| Exit: back button     | `PopScope(canPop: !_isSelecting)` — clears selection, stays on screen     |
| Exit: second long-press | Exits multi-select (second long-press on any cell = exit)              |

### 4. FavouriteSwipeScreen — Swipe-Through View (FAV-03, FAV-04)

Reuses `flutter_card_swiper` pattern from `CardSwipeScreen`. Shows all favourites starting
at the card matching the `favouriteId` route param.

| Property        | Value                                                                     |
|-----------------|---------------------------------------------------------------------------|
| Cards source    | Full `FavouritesNotifier` list, sorted newest first                       |
| Initial card    | Index of `favouriteId` in the sorted list                                 |
| Delete action   | `IconButton(Icons.delete_outline)` in `AppBar` actions                    |
| Delete icon color | `AppColors.error`                                                       |
| Delete tap      | Immediate Hive remove → Snackbar with Undo (D-08)                        |

### 5. Favourites Filter Bottom Sheet (FAV-07, D-11)

Reuses `FilterChip` + `Wrap` pattern from `FilterSettingsScreen`.

| Property          | Value                                                                         |
|-------------------|-------------------------------------------------------------------------------|
| Trigger           | Filter icon button (`Icons.filter_list`) in default app bar actions           |
| Sheet background  | `AppColors.surface`                                                           |
| Sheet padding     | `EdgeInsets.all(AppSpacing.md)` (horizontal + vertical)                       |
| Sections          | Colour, Type, Rarity — same labels as Phase 2                                 |
| Chip style        | Identical to `FilterSettingsScreen`: `FilterChip`, `showCheckmark: false`,    |
|                   | `selectedColor: AppColors.primary.withValues(alpha: 0.2)`                     |
| Chip spacing      | `spacing: AppSpacing.xs, runSpacing: AppSpacing.xs`                           |
| Active filters    | Chips show active selection state matching provider state                     |
| Reset             | Text button "Clear Filters" — `AppColors.error` foreground                    |
| Filter persistence| In-memory only, `autoDispose` — resets when tab is left (D-10)               |

### 6. Snackbar Patterns

| Trigger              | Message copy                        | Action label | Duration |
|----------------------|-------------------------------------|--------------|----------|
| Card saved (FAV-01)  | "Saved to Favourites"               | none         | ~2s      |
| Single delete (FAV-04)| "{card name} removed"              | "Undo"       | 3s       |
| Batch delete (FAV-04)| "{N} cards removed"                 | "Undo"       | 3s       |
| Already saved (guard)| no Snackbar shown (silent no-op)    | n/a          | n/a      |

Undo action label color: `AppColors.primary` (via `SnackBarTheme` or explicit `SnackBarAction.textColor`).
Always call `ScaffoldMessenger.of(context).clearSnackBars()` before `showSnackBar()`.

## Interaction States

Each async-capable surface must implement all required states.

### FavouritesScreen (grid)

| State    | Trigger                                  | UI                                                                 |
|----------|------------------------------------------|--------------------------------------------------------------------|
| Loading  | Provider not yet resolved                | `Skeletonizer` shimmer over placeholder 3-column grid             |
| Empty    | `filteredFavourites` list is empty AND no filter active | Empty state widget: icon + heading + body copy    |
| Empty (filtered) | List empty AND filter is active  | "No cards match your filters." + "Clear Filters" text button      |
| Success  | List has 1+ items                        | `SliverGrid` with `_FavouriteGridCell` items                       |
| Error    | Provider throws                          | Error widget with retry button (consistent with Phase 1/2 pattern)|

### FavouriteSwipeScreen

| State    | Trigger                                  | UI                                                                 |
|----------|------------------------------------------|--------------------------------------------------------------------|
| Loading  | Provider not yet resolved                | `Skeletonizer` shimmer over card-shaped placeholder                |
| Empty    | Favourites list is empty (all deleted)   | Navigate back to `FavouritesScreen` automatically                  |
| Success  | Card available at seeked index           | Full card swipe view with delete button in AppBar                  |

### Bookmark Button (on CardSwipeScreen)

| State    | Trigger                                  | UI                                                                 |
|----------|------------------------------------------|--------------------------------------------------------------------|
| Unsaved  | `isFavourite(card.id)` is false          | `Icons.favorite_border`, `AppColors.onBackground`, tappable       |
| Saved    | `isFavourite(card.id)` is true           | `Icons.favorite`, `AppColors.error`, disabled (`onPressed: null`) |

## Accessibility Notes

- All `IconButton` widgets must have a `tooltip` parameter:
  - Bookmark unsaved: `tooltip: 'Save to Favourites'`
  - Bookmark saved: `tooltip: 'Already saved'`
  - Delete in swipe: `tooltip: 'Remove from Favourites'`
  - Filter: `tooltip: 'Filter Favourites'`
- `Semantics` wrapper on grid cells: `label: '{card.name}'` so screen readers can identify cards.
- Multi-select checkmark overlay: add `Semantics(selected: true, label: '{card.name}, selected')`.
- Minimum touch target 44px — Flutter `IconButton` default (48px) satisfies this.
- Color is never the sole differentiator for state: bookmark uses icon shape change (border vs
  filled) in addition to color; selected grid cells use both overlay tint AND checkmark icon.
