**Class structure:**
`FilterSettingsScreen` extends `ConsumerStatefulWidget` (needs a `TextEditingController` for
the preset name field). State class: `_FilterSettingsScreenState extends ConsumerState<FilterSettingsScreen>`.
No `setState` — all state in Riverpod providers.

**Layout — SingleChildScrollView wrapping a Column with sections:**

1. **Preset chip row** (shown only when `presets.isNotEmpty`):
   - Section header: `Text('Presets', style: Theme.of(context).textTheme.titleMedium)`
   - `SingleChildScrollView(scrollDirection: Axis.horizontal)` containing a `Row` of `InputChip`s
   - Each chip label renders the dirty-state `*` suffix (D-12):
     ```dart
     label: Text(
       notifier.activePresetName == preset.name
           ? '${preset.name}*'
           : preset.name,
     )
     ```
     The `*` appears when `activePresetName` matches this preset's name, signalling that the
     user loaded this preset and has since modified a filter (because any mutation clears
     `activePresetName` — so the `*` is visible from load until the first mutation).
   - Each chip `onPressed`: call `loadPreset` with the preset name, then navigate:
     ```dart
     onPressed: () {
       ref.read(filterSettingsNotifierProvider.notifier)
           .loadPreset(preset.settings, presetName: preset.name);
       context.go(AppRoutes.discovery);
     }
     ```
   - Each chip has `deleteIcon: const Icon(Icons.close, size: AppSpacing.md)` and `onDeleted: () => presetsNotifier.delete(preset.name)`
   - Chips use `backgroundColor: AppColors.surfaceContainer`, `labelStyle: TextStyle(color: AppColors.onBackground)`

2. **Colour toggles** (FILT-01, D-01 through D-04):
