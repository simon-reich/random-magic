## Spacing Scale

Source: `lib/core/constants/spacing.dart` (VERIFIED)

| Token | Value | Usage |
|-------|-------|-------|
| AppSpacing.xs | 4px | Icon gaps, badge inner padding (vertical) |
| AppSpacing.sm | 8px | Badge inner padding (horizontal), chip gaps, row spacing within a section |
| AppSpacing.md | 16px | Section content padding (horizontal), standard element margin |
| AppSpacing.lg | 24px | Section header spacing, separator to next section |
| AppSpacing.xl | 32px | Between major content sections (oracle → set info → prices → legalities) |
| AppSpacing.xxl | 48px | Bottom padding below legalities (above nav bar) |

Exceptions:
- SliverAppBar `expandedHeight`: fixed at 440px (derived from card aspect ratio 63/88 at ~315px width — see D-06). This is a layout constant, not a spacing token; declare it as `_kDetailArtworkHeight = 440.0` in `card_detail_screen.dart`.
- FAB touch target: 56px (Flutter Material default for `FloatingActionButton`) — do not override.
- Tap target for card image (`GestureDetector`): full card face widget — no minimum override needed (covers entire artwork area, well above 48px minimum).

## Typography

Source: `lib/core/theme/app_theme.dart` TextTheme (VERIFIED). All sizes are Flutter SDK defaults for Material 3 unless specified.

| Role | Flutter TextStyle | Approx Size | Weight | Line Height | Usage in Phase 4 |
|------|-------------------|-------------|--------|-------------|-------------------|
| Body | `bodyMedium` | 14sp | 400 (regular) | 1.5 | Oracle text, flavour text, set info values, price values |
| Label | `labelSmall` | 11sp | 400 (regular) | 1.4 | Legality badge text, section captions |
| Title | `titleMedium` | 16sp | 500 (medium) | 1.3 | Section headers (Prices, Legalities, Set Info) |
| Heading | `headlineSmall` | 24sp | 600 (semibold) | 1.2 | Card name in collapsed AppBar title; screen heading |

Weights in use: regular (400) and semibold (600) only. Weight 500 (`titleMedium`) is the single concession for section headers — acceptable within "2-weight" intent as it maps to `w500` in existing `AppTheme`.

Flavour text: `bodyMedium` with `AppColors.onSurfaceMuted` (italic style via `TextStyle(fontStyle: FontStyle.italic)`). Hidden entirely when `card.flavorText == null` — no empty gap.

## Color

Source: `lib/core/theme/app_theme.dart` `AppColors` (VERIFIED)

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `AppColors.background` | #1A1D2E | Screen background, `CustomScrollView` background, collapsed AppBar background |
| Secondary (30%) | `AppColors.surface` | #252840 | Section card containers (oracle text block, set info block, prices block, legalities block) |
| Surface elevated | `AppColors.surfaceContainer` | #2F3250 | Inner chips, legality badge background fill (at 15% opacity over badge border color) |
| Accent (10%) | `AppColors.primary` | #C9A84C | Flip FAB background, `labelLarge` text, section divider accent line (optional) |
| Destructive | `AppColors.error` | #CF6679 | `Banned` legality badge border + text; no destructive actions in this phase |
| Muted text | `AppColors.onSurfaceMuted` | #9E9EAE | Secondary text, flavour text, `Not Legal` + `Restricted` badge color, section captions |
| New: Legal green | `AppColors.legal` | #4CAF50 | `Legal` legality badge border + text only — must be added to `app_theme.dart` |

Accent (`AppColors.primary` / gold) reserved for:
- Flip FAB button background
- `labelLarge` text style (already set in `AppTheme`)
- Active state indicators if any (none anticipated in this phase)

`AppColors.legal` is a new constant required by this phase. Add to `AppColors` in `lib/core/theme/app_theme.dart` before implementing `_LegalityRow`. Using a local `Color(0xFF4CAF50)` inline is explicitly forbidden by CLAUDE.md §No magic numbers.

### Legality Badge Color Map

| Status string | Badge color token | Badge label |
|---------------|-------------------|-------------|
| `'legal'` | `AppColors.legal` (#4CAF50) | Legal |
| `'banned'` | `AppColors.error` (#CF6679) | Banned |
| `'restricted'` | `AppColors.primaryVariant` (#F0A500) | Restricted |

## Screen Layout Contract

Source: CONTEXT.md D-05, D-06, D-07 (LOCKED)

### Structure

```
Scaffold
└── CustomScrollView
    ├── SliverAppBar (expandedHeight: 440.0, pinned: true)
    │   └── FlexibleSpaceBar
    │       ├── title: Text(card.name)  — visible in collapsed state
    │       └── background: _CardArtwork(imageUrl)  — large format image
    ├── SliverPadding (horizontal: AppSpacing.md)
    └── SliverList
        ├── _OracleTextSection          — oracle text (bodyMedium)
        ├── _FlavorTextSection          — conditional (hidden if null)
        ├── _SetInfoSection             — set name / collector number / release date
        ├── _PricesSection              — USD / USD Foil / EUR rows
        └── _LegalitiesSection          — Standard / Modern / Legacy / Commander rows
└── floatingActionButton: flip FAB (only when card.cardFaces != null)
```

Section order is locked (D-07). Dividers between sections: `Divider(color: AppColors.surfaceContainer, height: AppSpacing.xl)` — provides visual breathing room without adding heavy borders.

### SliverAppBar

- `expandedHeight`: 440.0 (constant `_kDetailArtworkHeight`)
- `pinned`: true — back button always accessible
- `stretch`: false
- `flexibleSpace`: `FlexibleSpaceBar` with `collapseMode: CollapseMode.parallax`
- Image: `CachedNetworkImage` using `card.imageUris.large ?? card.imageUris.normal` — guard for empty URL before passing
- `FlexibleSpaceBar.title`: `Text(displayName)` — appears in both collapsed and expanded; Flutter's `FlexibleSpaceBar` handles opacity automatically on collapse
- AppBar background color: `AppColors.background` (matches `AppBarTheme` in `AppTheme`)

### Flip FAB

- Shown only when `card.cardFaces != null`
- `FloatingActionButton` with `Icons.flip` icon
- Background: `AppColors.primary` (gold)
- Foreground icon: `AppColors.background` (dark navy — contrast on gold)
- `tooltip`: `'Show back face'` when `_showBack == false`; `'Show front face'` when `_showBack == true`
- On tap: `setState(() => _showBack = !_showBack)`

### Flip Behavior (D-04)

When `_showBack` toggles:
- Swap: image URL, card name, type line, oracle text
- Keep fixed: mana cost (always front face), prices, format legalities
- No flip animation required — simple `setState` rebuild

## Interaction Contract

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Navigate to detail | Tap card image in `CardSwipeScreen` or `FavouriteSwipeScreen` | `context.go('/card/${card.id}', extra: card)` |
| Back navigation | Tap AppBar back arrow or system back | `context.pop()` |
| Flip to back face | Tap Flip FAB | `setState(() => _showBack = true)`; image + name + typeLine + oracleText update; FAB tooltip changes |
| Flip to front face | Tap Flip FAB again | `setState(() => _showBack = false)`; all fields revert |
| Scroll | Drag up | `SliverAppBar` collapses; card name appears in AppBar title area |
| Card image load | Automatic on screen open | `CachedNetworkImage` with Skeletonizer shimmer placeholder; error icon on failure |

Flip FAB is hidden (set to `null`) for single-faced cards — `card.cardFaces == null`.

## Error & Loading States

| State | Trigger | UI |
|-------|---------|-----|
| Artwork loading | `CachedNetworkImage` in-flight | Skeletonizer shimmer (dark mode, inherits `SkeletonizerConfigData.dark()` from `AppTheme`) |
| Artwork error | `CachedNetworkImage` fails | `Icon(Icons.broken_image_outlined, color: AppColors.onSurfaceMuted)` centered in artwork area |
| Missing `extra` / null cast | GoRouter `extra` is not `MagicCard` | Full-screen error widget: icon + message + Back button (see Copywriting Contract) |
| Null price field | `card.prices?.usd == null` etc. | Show string `'N/A'` in price value column — no special widget |
| Null flavour text | `card.flavorText == null` | Section omitted entirely — no gap, no placeholder |

No loading spinner for screen-level state — the card data arrives synchronously via GoRouter `extra`. Only the artwork image has an async loading state.
